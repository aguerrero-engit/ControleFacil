      <h2 class="title">
      <a href="/node/2730">SpringSource Releases Hyperic 4.4</a>
    </h2>
  
  
  
      <div class="submitted">
      Submitted by Adam Fitzgerald on Wed, 2010-08-04 11:35    </div>
  
      <div class="taxonomy"> in <ul class="links inline"><li class="taxonomy_term_4 first last"><a href="/taxonomy/term/4" rel="tag" title="News and announcements
">News and Announcements</a></li>
</ul></div>
  
  <div class="content">
    <p>The SpringSource Hyperic product team is pleased to <a href="http://www.springsource.com/newsevents/hyperic-44-application-monitoring-virtual-infrastructure">announce</a> the general availability of <a href="http://www.springsource.com/products/systems-management">Hyperic 4.4</a>, the leading platform for custom web application monitoring and performance management in physical, virtual, and cloud environments. The <a href="http://www.hyperic.com/blog/announcing-hyperic-4-4-dramatically-simplified-monitoring-of-virtualized-application-infrastructure/">Hyperic blog</a> describes most of the new features included in the release, which include:</p>
<ul>
<li><strong>Rapid Diagnosis of Virtualized Application Performance Problems:</strong> Hyperic 4.4 now provides visibility into all application infrastructure layers including virtualization, enabling operators to determine whether the root cause of issues lies within an application, its guest operating system, or its ESX host. </li>
<li><strong>Automatic Maintenance of Application Infrastructure Inventory:</strong> Hyperic 4.4 automatically discovers ESX hosts, virtual machines, and guest operating systems within minutes of their launch, and presents them in a unified topology so users can see which application components are running on which ESX hosts. It even tracks applications when they move to new virtual machines via vMotion.</li>
<li><strong>Virtualization-Aware Alerting:</strong> Hyperic 4.4 can distinguish between when a guest operating system has shut down unintentionally, and when it has been intentionally powered down or suspended. This prevents false alarms and enables system administrators to elastically scale their application infrastructure without triggering alert storms.</li>
</ul>
<p>Check out the <a href="http://SpringSource.com/hyperic44" target="_blank">Hyperic 4.4 demo</a>, download <a href="http://www.springsource.com/downloads/hyperic-hq-enterprise-downloads" target="_blank">Hyperic Enterprise 4.4</a> or <a href="http://www.springsource.com/downloads/hyperic-open-source-downloads" target="_blank">Hyperic Open Source 4.4</a>, and dive into the new documentation on <a href="http://support.hyperic.com/display/DOC/Monitoring+vSphere+Components" target="_blank">monitoring vSphere</a>.</p>
  </div>
